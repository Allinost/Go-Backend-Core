# P0 — 发布前整改清单

开发阶段为方便调试放宽了多项限制，发布到生产环境前必须整改。

---

## 安全加固（严重）

| # | 检查项 | 开发期现状 | 发布要求 | 涉及文件 |
|---|--------|-----------|---------|----------|
| 1 | JWT 密钥 | `change-me-in-production` | 使用随机 256 位密钥，通过环境变量或 Docker Secret 注入 | `config.yaml` |
| 2 | 数据库密码 | 明文硬编码 `root` | 使用 Docker Secret 或环境变量 `APP_DATABASE_MYSQL_MAIN_PASSWORD` | `config.yaml`, `docker-compose.yml` |
| 3 | CORS 策略 | `Allow-Origin: *` 允许所有来源 | 限定特定域名列表，禁止通配符 | `internal/middleware/cors.go` |
| 4 | 限流策略 | 内存滑动窗口（进程重启即丢失） | 改用 Redis 分布式限流，避免单点绕过 | `internal/middleware/ratelimit.go` | ✅ 已实现：`RedisRateLimit` + `Limiter` 接口，内存/Redis 可切换 |
| 5 | 敏感字段加密 | 配置中明文存储 | 启用 `enc:aes-gcm:base64:` 加密方案，内存中解密 | `config.yaml`, `internal/services/crypto/` |
| 6 | Gin 运行模式 | `debug` | 改为 `release`，关闭 debug 路由和详细错误栈 | `config.yaml` |
| 7 | 错误详情泄露 | 非 release 模式下响应体携带 `detail` 字段暴露内部错误 | 确保 `server.mode: release` 隐藏错误详情 | `config.yaml`, `internal/pkg/response/response.go` |
| 8 | S0 调试路由暴露 | `/api/v1/s0/echo` 返回完整请求头，可能泄露 Authorization 等敏感字段 | 生产环境移除 S0 模块或限制仅内网访问 | `internal/modules/s0/module.go` |
| 9 | Secrets 明文存储 | `config.yaml` 密码字段留空，通过 `.env` 注入 | 生产环境改用 Docker Secrets 或环境变量注入，见 `docs/release/P0-密钥与Secrets管理.md` | `.env`, `config.yaml`, `internal/config/config.go` |

## 配置与环境隔离（高）

| # | 检查项 | 开发期现状 | 发布要求 | 涉及文件 |
|---|--------|-----------|---------|----------|
| 1 | 配置文件挂载 | 本地文件直接读取 | 通过 Docker volume 挂载 `config.yaml`，禁止打包进镜像 | `Dockerfile`, `docker-compose.yml` |
| 2 | 多环境分离 | 单文件 | 按环境拆分 `config.local.yaml` / `config.prod.yaml`，由 `APP_ENV` 切换 | `internal/config/config.go` |
| 3 | 环境变量覆盖 | 仅部分字段支持 | 所有敏感字段必须通过环境变量注入，配置文件中留空 | `internal/config/config.go` | ✅ 配置校验: `RegisterValidator` 可注册检查 |
| 4 | 端口暴露 | 直接暴露 29090 | 前端加 Nginx/Caddy 反向代理，由反向代理处理 TLS 终结 | `docker-compose.yml`, `deploy/` |

## 数据访问层（严重）

| # | 检查项 | 开发期现状 | 发布要求 | 涉及文件 |
|---|--------|-----------|---------|----------|
| 1 | 数据库连接加密 | 无 TLS，明文传输 | 启用 TLS/SSL（MySQL 和 Redis 均需），配置证书路径 | `internal/database/`, `config.yaml` |
| 2 | 数据库密码来源 | `.env` 文件 | 改用 Docker Secrets (`/run/secrets/`) 或 Vault，禁止文件明文 | `internal/database/mysql/mysql.go`, `docker-compose.yml` |
| 3 | 数据库连接熔断 | 无熔断，故障时直接 hang | 集成 circuit breaker（gobreaker），数据库故障时快速失败 | `internal/database/manager.go` |
| 4 | 数据库热加载 | 不支持配置变更热加载 | 实现 `Reload()` 方法，配置变更后零停机重建连接池 | `internal/database/manager.go` |
| 5 | PostgreSQL / MinIO / RustFS TLS | 连接无加密 | 启用 TLS/SSL，配置证书路径 | `internal/database/postgres/`, `internal/database/minio/`, `internal/database/rustfs/` |

> 详细清单见 `docs/release/P0-统一数据访问层生产就绪清单.md`

## 快速验证命令

```bash
# 检查是否使用了默认密钥
grep -r "change-me-in-production" config.yaml

# 检查 CORS 是否限制来源
grep "Access-Control-Allow-Origin" internal/middleware/cors.go

# 检查当前运行模式
grep "mode:" config.yaml

# 检查是否有明文密码
grep -E "password:\s+\w" config.yaml
```
