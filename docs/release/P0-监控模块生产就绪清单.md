# P0 — 监控模块生产就绪清单

> 当前 `internal/services/monitor/` 为快速迭代开发所做的妥协，**生产上线前必须修复**。

---

## 已实现的功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 健康检查聚合器 | ✅ | Register / Remove / Check（并发执行）/ IsHealthy |
| 主动轮询 | ✅ | StartPolling / Stop / LastResult，可配置间隔，支持 context 取消 |
| 超时保护 | ✅ | 每个 check 独立 5s 超时，context.WithTimeout |
| 状态变更通知 | ✅ | OnStatusChange 回调，pollOnce 自动检测 up/down 变化 |
| Prometheus 指标注册 | ✅ | NewCounter / NewGauge / NewHistogram + RegisterOrReuse 去重 |
| 指标去重保护 | ✅ | RegisterOrReuse 自动处理 AlreadyRegisteredError |
| `/metrics` HTTP 端点 | ✅ | promhttp.HandlerFor 标准格式 |
| Go runtime 指标 | ✅ | goroutines / mem_alloc / mem_sys / gc_count（GaugeFunc 自动采集） |
| s0/health 集成 | ✅ | HealthChecker 挂载到 `GET /api/v1/s0/health`，自动注册 database 健康检查 |
| HTTP 请求指标中间件 | ✅ | `internal/middleware/metrics.go` 自动采集请求量/延迟/状态码分布 |
| DB 连接池指标 | ✅ | `RegisterDBPoolMetrics` 采集连接池使用率（open/in_use/idle） |
| 自定义 Collector | ✅ | `dbPoolCollector` 演示了如何实现自定义 prometheus.Collector |

## 开发阶段放宽的条件（上线前需修复）

### 1. 健康检查缺少状态变更通知

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **状态变更事件** | 无 | 健康状态从 up→down 或 down→up 时通过 eventbus 发送事件 |

> ✅ `StatusChangeHandler` 回调机制已添加。通过 `OnStatusChange(handler)` 注册，`pollOnce()` 自动检测并触发。

### 2. 指标命名规范校验

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **命名校验缺失** | 无自动校验 | 实现指标名称格式校验（仅允许 `[a-zA-Z_:][a-zA-Z0-9_:]*`） |

### 3. 主动轮询集成未覆盖所有模块

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **轮询仅配置驱动** | s0 通过 `cfg.Config.Watch` 控制轮询开关 | 所有模块启动时按需注册定期检查 |

### 4. DB 连接池指标未接入实时数据

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **statsFunc 注入** | 由调用方提供 statsFunc，无自动采集 | 在 s0 或 database 模块自动注入真实连接池数据 |
