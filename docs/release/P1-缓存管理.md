# P1 — 缓存管理生产就绪清单

> 当前 `internal/services/cache/` 为快速迭代开发所做的妥协，**生产上线前必须修复**。

---

## 已实现的功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 二级缓存架构 | ✅ | L1 内存缓存 + L2 Redis 缓存，自动回源（Get）和回写（Set） |
| 通用接口 | ✅ | Cache 接口定义（Get/Set/Delete/Exists/Clear/Close） |
| 防缓存穿透 | ✅ | 空值缓存（NilValue），可配置空值 TTL |
| 防缓存雪崩 | ✅ | TTL 随机抖动（Jitter），均匀分布过期时间 |
| 防缓存击穿 | ✅ | singleflight 并发控制，同 key 只允许一个请求回源 |
| TTL 过期 | ✅ | L1/L2 独立 TTL + 随机 Jitter |
| 分布式支持 | ✅ | L2 Redis 原生支持分布式，shared 模式可对接已配置的 DB Redis |
| 可配置 | ✅ | 通过 config.yaml 配置缓存命名空间（namespace）、Redis 连接、TTL、Jitter |
| 全局管理器 | ✅ | `cache.InitAll(cfg)` 类似 `database.InitAll`，`cache.Get(name)` 获取实例，`cache.CloseAll()` 统一关闭 |
| Prometheus 指标注入 | ✅ | `InitAll` 自动注入 `metricsRegistry("go_backend_core", "cache")`，注册 cache_hits/misses/sets/evictions 四个 counter |
| 批量操作 | ✅ | `GetMany(keys)` 批量获取原始字节，`SetMany(items)` 批量设置 |
| L1 淘汰策略 | ✅ | 近似 LRU（lastHit 单调递增 clock），非随机淘汰 |
| 上下文感知 | ✅ | `Get` 先检查 `ctx.Err()`，返回上下文取消/超时错误 |
| 统计接口 | ✅ | `Stats()` 返回 L1 条目数、命中/未命中总数 |
| 分布式锁 | ✅ | `TryLock(ctx, key, ttl)` / `Unlock(ctx, key)`，基于 Redis SET NX EX |
| 预热机制 | ✅ | `Warmup(ctx, loader)` 启动时预加载热点数据到 L1+L2 |
| L1 失效通知 | ✅ | `PublishInvalidation(ctx, key)` 删除 L1+L2 + Redis PubSub 通知 |
| 慢日志 | ✅ | `Options.SlowThreshold`（默认 100ms），超时自动 `log.Printf` |
| 基准测试 | ✅ | BenchmarkSet / BenchmarkGet_Hit / BenchmarkGet_Miss / BenchmarkGetMany / BenchmarkTryLock |
| Makefile 竞态检测 | ✅ | `make test-race`（需 CGO_ENABLED=1），`make test-bench` |
