## 已修复的问题

### ✅ 1. 错误码双重维护 — 已修复

**做法**: `internal/pkg/errors/code.go` 的常量改为引用 `pkg/apierror` 定义：
```go
const (
    CodeSuccess    = apierror.CodeSuccess
    CodeParamErr   = apierror.CodeParamErr
    // ...
)
```
`CodeMsg()` 委托到 `apierror.CodeMsg()`。`codeMessages` map 从内部移除，`codeHTTPStatus` 仍保留在内部（公开包不需要 HTTP 映射）。

**验证**: `go test ./internal/pkg/errors/...` 全部通过。

### ✅ 2. 分页未对接数据库查询 — 已修复

**做法**: 新增 `pkg/pagination/query.go`，提供：
- `CursorQuery` 结构体（Column / Cursor / Direction / Limit）
- `BuildCursorQuery()` — 生成 `WHERE col > ? ORDER BY col ASC LIMIT ?` 等 SQL 子句
- `BuildOffsetQuery()` / `BuildPageQuery()` — Offset/Page 模式的 SQL 构建
- 前向(ASC) / 后向(DESC) 翻页支持
- 字符串和数值游标均支持

**验证**: 9 个新增测试用例，`go test ./pkg/pagination/...` 全部通过，覆盖率 100%。

### ✅ 3. 公开 API 稳定性 — 已修复

**做法**: 
- `pkg/apierror` 和 `pkg/pagination` 添加 package 级别 doc 注释标记 `v1alpha1`
- 文档注明 "在 v1 之前不保证向后兼容"
