# P0 — 事件总线生产就绪清单

> 当前 `internal/services/eventbus/` 为快速迭代开发所做的妥协，**生产上线前必须修复**。

## 已实现的功能

| 功能 | 状态 | 说明 |
|------|------|------|
| Event 结构体 | ✅ | ID / Topic / Source / Timestamp / Payload |
| EventBus 接口 | ✅ | Publish / Subscribe / Unsubscribe |
| 本地同步模式 | ✅ | 同进程内存调用，支持通配符 topic（`*` 后缀） |

## 开发阶段放宽的条件（上线前需修复）

### 1. Redis Stream 未生产就绪

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **断线重连** | go-redis 默认重连，但无自定义退避策略 | 实现重连回调 + 指数退避 + 重连期间事件缓存 |
| **消费者组监控** | 无 | 实现消费者组健康检查（pending 消息数 / 消费者活跃状态） |
| **消息回溯 API** | `Consume` 仅消费 `>`（新消息） | 支持从指定 ID 开始消费（`$`, `0`, 具体 ID） |
| **死信持久化** | 仅内存存储，重启丢失 | 死信消息写入 Redis Stream 死信 topic 或数据库表 |

> ✅ 断线重连: `Consume` 内嵌指数退避（500ms 起，最大 30s，最多 5 次重试）
> ✅ 死信持久化: 新增 `RedisDeadLetterQueue`（基于 Redis Stream），实现 `DeadLetterStore` 接口

### 2. 缺少异步发布

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **PublishAsync** | 接口设计中有 `PublishAsync` 但未实现 | 实现异步发布：goroutine + channel + 批量 flush |

### 3. 本地模式无持久化

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **消息丢失** | 本地模式纯内存，重启后所有历史消息丢失 | 本地模式仅用于开发/测试，生产必须使用 Redis Stream |
| **并发处理** | 多个订阅者串行执行 | 支持并发分发（goroutine per subscriber），可配置 |

### 4. 序列化兼容性

| 问题 | 现状 | 上线要求 |
|------|------|----------|
| **protobuf 预留** | 设计文档要求考虑 protobuf 兼容性但未实现 | Redis Stream 模式下 Payload 改为 `[]byte`（JSON 序列化 / protobuf 二选一），后续模块拆分时无缝切换 |
