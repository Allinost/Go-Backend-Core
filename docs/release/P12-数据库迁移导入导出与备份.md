# P12 — 数据库迁移（导入导出）与备份

---

## 已实现的功能

### 数据导出（Dump）

| 功能 | 状态 | 说明 |
|------|------|------|
| JSON 格式导出 | ✅ | 表名 → 行数组，支持 `_schema` 元信息 |
| CSV 格式导出 | ✅ | 表头行 + 数据行，含 `-- TABLE: xxx` 标记 |
| SQL 格式导出 | ✅ | `CREATE TABLE` + `INSERT INTO` 语句 |
| 仅导出结构 | ✅ | `SchemaOnly: true` 跳过数据 |
| 批量流式导出 | ✅ | `BatchSize` 控制每批行数，避免 OOM |
| WHERE 条件过滤 | ✅ | 支持按表名指定过滤条件 |

### 数据导入（Restore）

| 功能 | 状态 | 说明 |
|------|------|------|
| JSON 导入 | ✅ | 自动匹配目标表列名 |
| CSV 导入 | ✅ | 从表头列名映射 |
| SQL 导入 | ✅ | 逐条执行 INSERT/CREATE 语句 |
| 自动检测格式 | ✅ | `detectFormat()` 根据内容判断 |
| 导入前清空 | ✅ | `Truncate: true` |
| 跳过已有数据 | ✅ | `SkipExisting: true` |

### 跨数据库迁移（Transfer）

| 功能 | 状态 | 说明 |
|------|------|------|
| MySQL → PostgreSQL | ✅ | 类型自动转换（`BIGINT`/`TEXT`/`JSONB`/`BOOLEAN` 等） |
| PostgreSQL → MySQL | ✅ | 自动创建目标表 |
| 批量传输 | ✅ | 流式读取 + 批量写入 |

### 备份与恢复

| 功能 | 状态 | 说明 |
|------|------|------|
| JSON 备份 | ✅ | Dump → 压缩 → 加密 → 写文件 |
| 压缩 | ✅ | gzip / zlib（复用 crypto 服务） |
| 加密 | ✅ | AES-256-GCM 密码加密（复用 crypto 服务） |
| 恢复 | ✅ | 读文件 → 解密 → 解压 → Restore |
| 备份列表 | ✅ | `ListBackups()` 列出 .meta.json |
| 备份删除 | ✅ | `DeleteBackup()` |

### Schema 迁移

| 功能 | 状态 | 说明 |
|------|------|------|
| `_migrations` 版本表 | ✅ | `Init()` 自动创建 |
| 已应用列表 | ✅ | `Applied()` 查询版本表 |
| 待定列表 | ✅ | `Pending()` 文件目录 vs 版本表 |
| 单步执行 | ✅ | `Apply(mig)` 原子化执行 |
| 批量执行 | ✅ | `ApplyAll()` 执行所有待定迁移 |
| 生成迁移文件 | ✅ | `CreateMigrationFile(name)` 自增编号 |
| 状态查询 | ✅ | `Status()` 返回总览 + 每个文件的详细状态 |

## 使用示例

### 导出数据

```bash
curl -X POST http://localhost:8080/api/v1/migrate/dump \
  -H 'Content-Type: application/json' \
  -d '{"format":"json","tables":["users","roles"]}' \
  -o dump.json
```

### 导入数据

```bash
curl -X POST http://localhost:8080/api/v1/migrate/restore \
  -F 'file=@dump.json'
```

### 跨库迁移

```bash
curl -X POST http://localhost:8080/api/v1/migrate/transfer \
  -H 'Content-Type: application/json' \
  -d '{"target_db_type":"postgres","tables":["users"],"create_table":true}'
```

### 创建加密备份

```bash
curl -X POST http://localhost:8080/api/v1/migrate/backup \
  -H 'Content-Type: application/json' \
  -d '{"tables":["users","orders"],"compress_algo":"gzip","encrypt_key":"my-secret-key","output_dir":"backups/"}'
```

### Schema 迁移

```bash
# 创建迁移文件
curl -X POST http://localhost:8080/api/v1/migrate/schema/create \
  -H 'Content-Type: application/json' \
  -d '{"name":"add email column"}'

# 查看待定迁移
curl http://localhost:8080/api/v1/migrate/schema/migrations

# 执行所有待定迁移
curl -X POST http://localhost:8080/api/v1/migrate/schema/migrate
```

## DBAdapter 接口

```go
type DBAdapter interface {
    Type() DBType                              // MySQL / Postgres
    Ping(ctx) error                            // 健康检查
    GetTables(ctx) ([]string, error)           // 表列表
    GetTableSchema(ctx, table) (*TableSchema, error)
    GetColumns(ctx, table) ([]ColumnInfo, error)
    SelectAll(ctx, table, columns, where, batchSize, fn) error  // 流式读取
    InsertBatch(ctx, table, columns, rows) error               // 批量写入
    Exec(ctx, sql, args...) error                              // 执行 SQL
    ExecRaw(ctx, sql) error                                    // 多语句 SQL
    Truncate(ctx, table) error
    CreateTableIfNotExists(ctx, schema) error
    Quote(name) string
}
```

现有实现：`MySQLAdapter`（基于 `*sql.DB`）、`PostgresAdapter`（基于 `*pgxpool.Pool`）。

## Schema 迁移文件命名规则

```
001_init.sql
002_create_users.sql
003_add_email_column.sql
```

格式：`<编号>_<描述>.sql`，编号自动递增，内容为纯 SQL DDL。
