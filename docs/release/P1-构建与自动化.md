# P1 — 构建与自动化

开发期手动 `go run`，发布后需统一构建流程和 CI 自动化。

---

## 构建脚本差异

| # | 检查项 | 开发期现状 | 发布要求 | 涉及文件 |
|---|--------|-----------|---------|----------|
| 1 | 调试符号 | debug 模式保留（`-gcflags="-N -l"`） | release 模式用 `-ldflags="-s -w"` 剥离，缩减体积 30%+ | `scripts/build.sh`, `Makefile` |
| 2 | CGO 依赖 | 开发期可开启 CGO | release 构建必须 `CGO_ENABLED=0` 确保静态编译，避免运行时 glibc 依赖 | `scripts/build.sh` |
| 3 | 跨平台编译 | 默认本机平台 | CI 中需指定 `GOOS=linux GOARCH=amd64` 确保镜像兼容 | `scripts/build.sh`, `.github/workflows/ci.yml` |
| 4 | 版本注入 | 无 | release 构建应通过 `-ldflags="-X main.Version=$(git describe)"` 注入版本号 | `scripts/build.sh` |

## 本地开发 vs CI 执行

| # | 检查项 | 开发期 | CI/发布 |
|---|--------|--------|---------|
| 1 | 测试运行 | `go test ./...` | 需加 `-race -coverprofile=coverage.out` 检测竞态和覆盖率 | `scripts/test.sh` |
| 2 | Lint 检查 | 可选执行 | 必须通过 `golangci-lint run` 才能合并 PR | `scripts/lint.sh` |
| 3 | 构建产物 | 本地二进制 | 发布需 `docker build` 推送镜像到 registry | `Makefile`, `.github/workflows/ci.yml` |
| 4 | Makefile 依赖 | 需本地安装 make | CI 中可直接调 `go build` 避免 make 依赖 | `Makefile` |

## 常用命令速查

```bash
# 开发调试
make run                        # debug 构建 + 启动
make test                       # 跑测试
make lint                       # 代码检查

# 发布构建
make release                    # 剥离调试信息
make docker-build               # 构建 Docker 镜像

# 清理
make clean

# 所有目标
make help
```

## 快速验证

```bash
# 对比 debug 和 release 体积
make build    && ls -lh server
make release  && ls -lh server

# 检查是否静态编译（不应显示 "not a dynamic executable"）
file server | grep -q "statically linked" && echo "静态编译 ✓"
```
