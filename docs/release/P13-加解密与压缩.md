## 字段级自动加解密

### 用法

在 GORM 模型 struct 中标记敏感字段：

```go
type User struct {
	ID       uint   `json:"id"`
	Username string `json:"username"`
	Email    string `json:"email" encrypt:"true"`      // 自动加解密
	Phone    string `json:"phone" encrypt:"true"`      // 自动加解密
	IDCard   string `json:"id_card" encrypt:"true"`    // 自动加解密
	Bio      string `json:"bio"`                        // 明文
}
```

加密：
```go
fe := crypto.NewFieldEncryptor(km)
fe.EncryptFields(&user)  // 仅 encrypt:"true" 字段被加密
```

解密：
```go
fe.DecryptFields(&user)  // 仅 encrypted 字段被解密
```

也支持切片和 `*string` 指针字段：
```go
fe.EncryptSlice(&users)
fe.DecryptSlice(&users)
```

### GORM 集成示例

```go
func (u *User) BeforeCreate(tx *gorm.DB) error {
    fe := crypto.NewFieldEncryptor(getKeyManager())
    return fe.EncryptFields(u)
}
func (u *User) AfterFind(tx *gorm.DB) error {
    fe := crypto.NewFieldEncryptor(getKeyManager())
    return fe.DecryptFields(u)
}
```

## 密钥版本化与轮换

### 密文格式

当前加密 `"secret@email.com"` → 存储 `v1:aabbccddee...`

- `v<ver>:` 前缀标识密钥版本
- 旧版本密钥无前缀 → 自动使用当前密钥解密（向后兼容）

### KeyManager

```go
// 初始版本: 只有 current key
km, _ := crypto.NewKeyManager("current_hex_key", nil)

// 轮换后: current 换新, old 保留旧密钥用于解密旧数据
km, _ := crypto.NewKeyManager("new_hex_key", map[int]string{
    1: "old_hex_key",
})
// currentVer 自动设为 2
// 加密 → "v2:...", 解密能识别 v1 和 v2
```

### 配置

```yaml
crypto:
  enabled: true
  master_key_hex: "新密钥hex"
  old_keys:
    1: "旧密钥hex"     # 密钥版本 → hex 密钥
```

### 手动轮换命令

```go
// 全表重加密: 逐行读 → 旧密钥解密 → 新密钥加密 → 写回
count, err := crypto.ReEncryptTable(db, fe, "users", &User{}, 100)

// 指定字段重加密 (更灵活)
count, err := crypto.ReEncryptRows(db, fe, "users", "id", []string{"email", "phone"}, 100)
```

## 文件加密

### 流式文件加密/解密

```go
// 加密大文件（逐块处理，不加载到内存）
err := crypto.EncryptFile("backup.sql", "backup.sql.enc", key)

// 解密
err := crypto.DecryptFile("backup.sql.enc", "backup.sql", key)

// 先压缩再加密
err := crypto.EncryptAndCompressFile("backup.sql", "backup.sql.enc", key)

// 先解密再解压
err := crypto.DecryptAndDecompressFile("backup.sql.enc", "backup.sql", key)
```

### 分片压缩

```go
c := crypto.NewSplitCompression(10*1024*1024, "", "shard")  // 10MB 每片
set, _ := c.Compress(largeData)                              // 返回 ShardSet
data, _ := c.Decompress(set)

// 序列化/反序列化 (JSON)
serialized, _ := c.Serialize(set)
set2, _ := c.Deserialize(serialized)
```

## 哈希

```go
// 内容哈希
h256 := crypto.SHA256Hash(data)
h512 := crypto.SHA512Hash(data)
h3   := crypto.BLAKE3Hash(data)

// 通用接口
h, _ := crypto.Hash(data, crypto.HashSHA256)

// 密码哈希 (Argon2id)
encoded, _ := crypto.HashPassword("user-password")
match, _   := crypto.VerifyPassword("user-password", encoded)
```

## 算法选择指南

| 场景 | 推荐算法 | 说明 |
|------|----------|------|
| 通用数据加密 | AES-256-GCM | 硬件加速，性能优异 |
| 移动端/低性能设备 | ChaCha20-Poly1305 | 无硬件加速时更快 |
| 文件加密 | AES-256-GCM + zstd/gzip | 先压缩后加密 |
| 密码存储 | Argon2id | 抗 GPU/ASIC 暴力破解 |
| 数据完整性 | SHA-256 / BLAKE3 | BLAKE3 性能远超 SHA-256 |
| 日志压缩 | zstd | 高压缩比 |
| Web 内容 | gzip | 浏览器广泛支持 |
