# S0 调试服务

调试与健康检查模块，**生产环境应移除或限制内网访问**。

**路径前缀**: `/api/v1/s0/`

---

## GET /ping

存活检查，无参数。

### 请求示例

```
GET /api/v1/s0/ping
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": "pong"
  }
}
```

---

## GET /health

各依赖服务健康状态聚合。

### 请求示例

```
GET /api/v1/s0/health
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok",
    "version": "v0.1.0"
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 整体状态 `ok` / `degraded` / `down` |
| version | string | 当前服务版本号 |

---

## GET /echo

请求回显，用于调试时查看请求信息。**⚠️ 会返回请求头，可能泄露 Authorization 等敏感字段。**

### 请求示例

```
GET /api/v1/s0/echo?foo=bar
X-Custom: test-value
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "method": "GET",
    "path": "/api/v1/s0/echo",
    "query": "foo=bar",
    "headers": {
      "X-Custom": ["test-value"]
    },
    "client_ip": "127.0.0.1"
  }
}
```
